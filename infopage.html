PhishingDetector - Information Page

What is PhishingDetector?

Phishing Detector is a project of the LMU Munich in cooperation with the Bundeswehr University, which was developed in the context of a master thesis. The aim is to test whether the spatial proximity of warnings has an influence on user behavior, and whether there are properties that improve or worsen this behavior. The master thesis is written by Florian Hartl, the supervisors are M.Sc. Felix Dietz and Dr. Pascal Knierim. The thesis will be titled "Development and Evaluation of a Browser Plugin To Prevent Social Engineering Attacks".

How does it work?
PhishingDetector scans the domain of every page you visit. Then it matches whether the domain is in a white- or blacklist of websites that we obtain from Mendeley. The origin of the files is a Mendeley Data Set. For websites that are not in these lists, we request an analysis from Virustotal.com for the domain. This site scans websites with about 80 virus scanners and returns the result. If more than one scan finds the website to be malicious, we create a warning. You can then decide for yourself in the pop-up window whether you trust the scan or classify the site as safe.

I am part of the Prolific study, but entered my Prolific ID incorrectly. What can I do?

When entering the ProlificID, make sure to enter all characters without errors so that we can assign the contributions to your account. If you mistyped the Prolific ID, please write us an email with the displayed ID in the popup AND your Prolific ID.

What data is shared with the developers?

As part of the study, we store data that the tool creates. For example, injections of the icon, scans of pages and behavior when displaying the injected icon. All data is transferred anonymously. In particular, web pages are also shortened to their domains, i.e. they are not transmitted in full (for example, for https://www.facebook.com/martin.fischer only www.facebook.com is stored). Should the functionality be disabled, no further data will be collected or shared. The last insertion of the PDIcon and the known Domains are always saved, but only transferred if the option is set. Under no circumstances will information be published in the thesis that could indicate the identity of users. You can also check out the Privacy Policy.

Do I have to leave all options enabled for the study?

For the study, the general functionality and the upload of data must be enabled. Background coloring can be enabled or disabled as desired. NOTE: In very rare cases (for example for a few computers that are administrated by your company) it can occur that the options reenable themselves. In that case please use Inkognito tabs when you don't want the plugin to work.

Why does the yellow icon appear some times and turns green after some time?

The yellow icon shows you, that we are not finished retrieving all the needed information. If a web page is not known by your browser's version of PhishingDetector, a virus scan is requested. If the page was never scanned before, we request a scan by Virustotal which initiates a scan by multiple detectors. After this scan the result is returned to the plugin. Until all this is processed, the icon's color will be yellow. You can retrieve information about the current status that will be updated once the scan results are ready.

Why does the scan not work on some pages?

On most public accessible pages the scan and the display of the icon should work without any problems. Login pages that are dynamically produced will not work with the inserted icon. On system pages and files that are open locally, no analysis is performed and the icon remains yellow. If the volume of requests is too high, it may be that temporarily no new analyses can be retrieved. This error will fix itself after 24 hours at the latest.

The popup in the status bar does not show any reasonable information?

In some cases, a page cannot be classified by PhishingDetector. Examples are:

  • The page is not fully loaded yet
  • The plugin is disabled
  • The page was not opened with the plugin enabled
  • The page is a system page
  • The page contains a locally openned document
