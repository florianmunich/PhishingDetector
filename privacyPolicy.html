Below, the imprint is stated.

What data is collected and how is it processed?
In general, we distinguish between data that the plugin processes locally for functionality and data that is shared with developers.
1) Data processed by the plugin (local only)
  • Every website visit is detected and processed by the plugin. When a domain is visited for the first time (or when 1000 other domains are visited in the meantime), a local comparison is made with a database. If no entry is found in this database, the domain (not the entire URL) is sent to Virustotal.com for a virus scan.
  • Domains already visited are saved locally
  • Installation date, last icon insertions and installation date are saved locally
  • If data sharing is enabled, all activities of the plugin are stored locally and then shared (see below)
2) Data shared by the plugin with developers (if enabled in the settings)
  • Installation date
  • Initialization on a web page, as well as leaving a web page
  • Result of a matching of a domain with our database or shortened result of virus scan
  • Activity data of hovering over the icon, as well as leaving or reloading a web page or tab, displaying the popup of the extension and changing settings
  • All web pages are stored and shared only in a shortened form (for example, for https://www.facebook.com/martin.fischer only www.facebook.com is stored)
The collected data will be uploaded either with a random ID, or the entered Prolific ID on a server of the Bundeswehr University and evaluated there in the context of the master thesis "Development and Evaluation of a Browser Plugin To Prevent Social Engineering Attacks". No personalized data will be published. Under no circumstances will information be published in the thesis that could indicate the identity of users. Only the authors of the thesis will have access to the data sets. They will not be published and will be destroyed after publication of the thesis.
